MedicalWebPage 구조 분석

검색엔진이 의료 정보를 더 잘 이해하도록 돕는 Schema.org 마크업의 구조를 시각적으로 설명합니다.

application/ld+json
"@context": "https://schema.org",
"@type": "MedicalWebPage",
"name": "대사증후군과 영양",
"description": "...",
"audience": {
"@type": "MedicalAudience",
"audienceType": "patient"
},
"lastReviewed": "2023-10-27",
"specialty": {
"@type": "MedicalSpecialty",
"name": "Nutrition"
},
"publisher": { ... }
@context

스키마 표준 정의

이 데이터가 Schema.org 표준을 따른다는 것을 검색엔진에 알립니다.

@type

MedicalWebPage

일반 웹페이지가 아닌 의학 정보 페이지임을 명시합니다. 구글이 의료 정보로 인식하게 하는 핵심입니다.

name

페이지 제목

검색 결과에 표시될 페이지의 공식 제목입니다.

description

내용 요약

페이지의 핵심 내용을 요약합니다. 검색 결과의 스니펫으로 사용될 수 있습니다.

audience

타겟 독자 (환자)

이 정보가 의료 전문가용인지, 일반 환자용(patient)인지 구분합니다.

lastReviewed

최종 검토일

의료 정보의 최신성을 보장하기 위해 언제 마지막으로 의학적 검토를 거쳤는지 명시합니다.

specialty

관련 전문 분야

해당 정보가 속한 의학 분야(예: 영양학, 심장내과 등)를 지정합니다.

publisher

발행 기관

정보의 신뢰도를 위해 발행 기관(병원) 정보를 포함합니다.

구조도 (Structure Map)

정확한 타입 명시

일반 WebPage가 아닌 MedicalWebPage를 사용하여 의료 정보임을 명확히 하세요.

최신성 유지

lastReviewed 속성을 통해 정보가 최신 의학 지식을 반영함을 증명하세요.

타겟 명확화

전문가용(Clinician)인지 환자용(Patient)인지 구분하여 검색 의도에 맞추세요.
